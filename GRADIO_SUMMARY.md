# Gradio 测试平台 - 功能总结

## 🎉 新增功能

我们为语音聊天服务添加了一个强大的 **Gradio 测试平台**，提供真实的音频测试环境。

## 📁 新增文件

1. **gradio_app.py** - Gradio 测试应用主程序（~350 行）
2. **README_GRADIO.md** - Gradio 平台详细使用文档
3. **QUICKSTART.md** - 更新了快速开始指南，突出 Gradio 测试

## 🚀 快速启动

### 第一步：启动后端服务

```bash
# 在终端 1
python -m app.main
```

### 第二步：启动 Gradio 测试平台

```bash
# 在终端 2
python gradio_app.py
```

### 第三步：访问界面

浏览器打开：`http://localhost:7860`

## ✨ 核心功能

### 1. 真实音频输入 🎤

- **麦克风录音**：直接在浏览器中录制音频
- **文件上传**：支持 WAV, MP3, OGG, FLAC 等格式
- **自动转换**：自动转换为 16kHz 单声道 PCM

### 2. 完整的处理流程展示 📊

- **状态信息**：显示处理状态、技能 ID、置信度等
- **ASR 识别**：显示语音识别的文本结果
- **LLM 回复**：显示智能回复的文本内容
- **TTS 音频**：播放和下载合成的语音

### 3. 友好的用户界面 🎨

- **现代化设计**：基于 Gradio Soft 主题
- **响应式布局**：自适应不同屏幕尺寸
- **实时反馈**：即时显示处理结果
- **错误提示**：清晰的错误信息

### 4. 便捷的测试工具 🛠️

- **服务健康检查**：一键检查后端服务状态
- **上下文配置**：支持自定义 JSON 上下文
- **清空功能**：快速清除输入
- **预定义示例**：快速测试固定指令

## 📦 技术实现

### 核心依赖

```python
gradio==4.44.0      # Web 界面框架
librosa==0.10.1     # 音频处理
soundfile==0.12.1   # 音频文件读写
websockets          # WebSocket 客户端
```

### 关键功能实现

1. **音频处理**
   ```python
   - librosa.load(): 加载音频文件
   - librosa.resample(): 重采样到 16kHz
   - 转换为单声道 PCM
   ```

2. **WebSocket 通信**
   ```python
   - 异步连接到后端服务
   - Base64 编码音频数据
   - 解析 JSON 响应
   ```

3. **界面组件**
   ```python
   - gr.Audio: 音频录制和播放
   - gr.Textbox: 文本输入输出
   - gr.Markdown: 状态展示
   - gr.Button: 操作按钮
   ```

## 🔧 配置和扩展

### 修改端口

在 `gradio_app.py` 的 `main()` 函数中：

```python
demo.launch(
    server_name="0.0.0.0",
    server_port=7860,  # 修改这里
    share=False,
)
```

### 启用公共链接

```python
demo.launch(
    share=True,  # 生成公共链接
    server_name="0.0.0.0",
    server_port=7860,
)
```

### 添加认证

```python
demo.launch(
    auth=("username", "password"),  # 添加认证
    server_name="0.0.0.0",
    server_port=7860,
)
```
